### Инструкция по выполнению домашнего задания

   1. Сделайте `fork` данного репозитория к себе в Github и переименуйте его по названию или номеру занятия, например, https://github.com/имя-вашего-репозитория/git-hw или  https://github.com/имя-вашего-репозитория/7-1-ansible-hw).
   2. Выполните клонирование данного репозитория к себе на ПК с помощью команды `git clone`.
   3. Выполните домашнее задание и заполните у себя локально этот файл README.md:
      - впишите вверху название занятия и вашу фамилию и имя
      - в каждом задании добавьте решение в требуемом виде (текст/код/скриншоты/ссылка)
      - для корректного добавления скриншотов воспользуйтесь [инструкцией "Как вставить скриншот в шаблон с решением](https://github.com/netology-code/sys-pattern-homework/blob/main/screen-instruction.md)
      - при оформлении используйте возможности языка разметки md (коротко об этом можно посмотреть в [инструкции  по MarkDown](https://github.com/netology-code/sys-pattern-homework/blob/main/md-instruction.md))
   4. После завершения работы над домашним заданием сделайте коммит (`git commit -m "comment"`) и отправьте его на Github (`git push origin`);
   5. Для проверки домашнего задания преподавателем в личном кабинете прикрепите и отправьте ссылку на решение в виде md-файла в вашем Github.
   6. Любые вопросы по выполнению заданий спрашивайте в чате учебной группы и/или в разделе “Вопросы по заданию” в личном кабинете.
   
Желаем успехов в выполнении домашнего задания!

### Задание 1


```
1. Проблемы производительности
Высокая нагрузка на базу данных

Проблема: Частые запросы к БД для одних и тех же данных

Решение: Кеширование результатов запросов снижает нагрузку на БД

Медленные вычисления

Проблема: Сложные расчеты, агрегации, обработка данных

Решение: Кеширование результатов вычислений

Долгие внешние API-вызовы

Проблема: Зависимость от медленных сторонних сервисов

Решение: Кеширование ответов внешних API

2. Проблемы масштабируемости
Пиковые нагрузки

Проблема: Внезапное увеличение трафика (например, распродажа, новостной всплеск)

Решение: Кеш поглощает пиковые нагрузки

Географическая задержка

Проблема: Пользователи из разных регионов получают данные с разной скоростью

Решение: Распределенный кеш (CDN, edge caching)

3. Проблемы доступности
Отказ базы данных

Проблема: При падении БД приложение перестает работать

Решение: Кеш может временно обслуживать запросы

Обслуживание БД

Проблема: Необходимость остановки БД для технических работ

Решение: Кеш обеспечивает работу приложения во время обслуживания

4. Проблемы стоимости
Дорогие операции

Проблема: Высокая стоимость вычислительных ресурсов или внешних сервисов

Решение: Кеширование снижает количество дорогостоящих операций

Масштабирование инфраструктуры

Проблема: Необходимость мощного железа для обработки пиковых нагрузок

Решение: Кеш позволяет использовать менее мощное оборудование

5. Проблемы пользовательского опыта
Долгая загрузка страниц

Проблема: Медленный отклик приложения

Решение: Кеширование ускоряет время ответа

Нестабильная работа

Проблема: Разная скорость работы в разное время суток

Решение: Кеш стабилизирует производительность

6. Конкретные примеры из практики
Электронная коммерция

Кеширование каталога товаров, цен, акций

Кеширование корзин покупок

Социальные сети

Кеширование ленты новостей, профилей пользователей

Кеширование счетчиков (лайки, просмотры)

Медиа-сайты

Кеширование статей, изображений, видео

Кеширование популярного контента

Аналитические системы

Кеширование отчетов и дашбордов

Кеширование результатов сложных запросов

7. Технические проблемы
Проблема N+1 запросов

Проблема: Множество отдельных запросов к БД

Решение: Кеширование связанных данных

Блокировки базы данных

Проблема: Конфликты при одновременном доступе

Решение: Кеш снижает количество операций записи

Кеширование - это компромисс между актуальностью данных и производительностью, 
который требует правильной стратегии инвалидации и обновления кеша.
```
